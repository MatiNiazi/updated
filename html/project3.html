My Project

Project Description

"MyArt" is an innovative web project that brings art enthusiasts closer to the fascinating world of art using data from the Art Institute of Chicago's API. This project features meticulously designed pages and components, seamlessly integrated with the API to provide users with a rich and immersive experience. From exploring artworks and exhibitions to discovering renowned artists, MyArt offers a user-friendly interface that enhances engagement and appreciation for art

Project Requirements

Functional Requirements

Artwork Display:

The application should display artworks from the Art Institute of Chicago's API, including images, descriptions, and artist information.

Exhibitions Display:

The application should display Exhibitions from the Art Institute of Chicago's API, including images, descriptions, and artist information.

products display:

The application should display products from the Art Institute of Chicago's API, including images, descriptions, and artist information.

Tours display:

The application should display Tours from the Art Institute of Chicago's API, including images, descriptions, and artist information.

Search Functionality:

Users should be able to search for artworks, exhibitions, and artists based on keywords or categories.

Responsive Design::

The UI should be responsive and accessible across various devices such as desktops, tablets, and smartphones.

Non Functional Requirements

Security:

Secure user authentication and authorization mechanisms to protect sensitive data
Regular security audits and updates to ensure system security.

Performance:

Fast loading times for both customer and admin interfaces.
Efficient database queries and caching mechanisms to improve performance.

Scalability:

Ability to handle a growing number of users, cars, and bookings without performance degradation.
Scalable database architecture to accommodate increasing data.

Reliability:

Ensure system uptime and availability for users and admins.
Implement backup and recovery mechanisms to prevent data loss.

Compatibility:

Support for multiple browsers and devices (desktop, mobile, tablet).
Ensure cross-browser compatibility and responsive design for a consistent user experience.

Documentation

Comprehensive documentation for users and administrators on how to use the system.
Code documentation for developers to understand and maintain the system effectively.

Software Model

Sequential Model

MyArt project adopts a hybrid software development model, blending elements of the waterfall and agile methodologies. It begins with structured requirements gathering and design, following a sequential approach akin to the waterfall model. However, it also incorporates agile practices such as iterative testing before deployment and making post-deployment changes based on user feedback. This hybrid approach allows for a well-planned initial design while remaining adaptable to evolving needs and continuous improvement.
